# 3.1 파트너십 협력 구조 (R&R)

---

## 3.1.1 역할 분담표

| 역할 | (주)티웨이브 (아임인) | 현지 디지털뱅크 |
|------|-------------------|--------------|
| **플랫폼** | SNBL API 제공, I-CSS AI 리스크 엔진 운영 | 현지 금융 라이선스 기반 에스크로 계좌 운영 |
| **데이터** | Alpha Credit 스코어링 생성·제공 | 계좌 개설 및 KYC/AML 처리 |
| **고객** | 스테이지 참여자 모집·운영 | 스테이지 완료 유저 대상 금융 상품 제공 |
| **리스크** | I-CSS로 부도 리스크 원천 차단 | 법적 에스크로 관리·자금 보호 |
| **수익** | 스테이지 수수료 (납입액의 1.1%) | 스테이지 완료 유저 대상 여수신 수익 |

---

## 3.1.2 API 연동 방식

아임인은 **화이트라벨(White-label) SaaS 방식**과 **API 연동 방식** 두 가지를 제공합니다.

### 옵션 A — 화이트라벨 임베드 (권장)
```
파트너 은행 앱 내부에 아임인 스테이지 기능 직접 임베드
  
  [파트너 은행 앱]
  ├── 기존 예금/대출/투자 메뉴
  └── [목돈 저축 클럽] ← 아임인 SNBL 화이트라벨
      (파트너 은행 브랜드로 표시)
```

**장점:** 유저가 파트너 은행 생태계에서 이탈 없이 경험 → MAU 증가

### 옵션 B — 외부 앱 연동 (딥링크)
```
아임인 독립 앱 ↔ 파트너 은행 계좌 연동
  
  납입금 → 파트너 은행 지정 계좌
  Alpha Credit → 파트너 은행 API로 공유
  스테이지 완료 → 파트너 은행 상품 추천 푸시
```

## 3.1.3 수익 배분 모델

| 수익원 | 아임인 몫 | 파트너 은행 몫 |
|--------|---------|--------------|
| 스테이지 플랫폼 수수료 (1.1%) | 전액 | — |
| Alpha Credit 기반 대출 실행 수수료 | 30~40% | 60~70% |
| 스테이지 완료 후 금융 상품 가입 수수료 | 20~30% | 70~80% |
| 예금 유치로 인한 NIM 개선 효과 | — | 전액 (은행 수혜) |

> **파트너 은행의 가장 큰 수혜는 '역마진 없는 예금 유동성'입니다. 이 자체로 연간 수십억~수백억 원의 비용 절감 효과가 발생합니다.**

---

## 3.1.4 규제 컴플라이언스 구조

아임인은 파트너 은행의 라이선스에 올라타는 B2B SaaS 구조이므로, 직접적인 금융 라이선스 없이 합법적으로 운영 가능합니다.

```
[규제 안전 구조]

  아임인 역할: IT 플랫폼 운영 (SaaS/API)
      ↓
  자금 보관·흐름: 파트너 은행 에스크로 계좌
      ↓
  법적 책임: 파트너 은행이 현지 금융 당국에 보고
      ↓
  결과: 유사수신·P2P 대출 규제 적용 없음
```

| 규제 우려 | 해결 방식 |
|---------|---------|
| 유사수신 의혹 | 자금이 파트너 은행 에스크로에 보관 → 아임인은 IT 플랫폼만 운영 |
| P2P 대출 규제 | 대출 실행 주체는 파트너 은행 → B2B2C 구조로 규제 마찰 없음 |
| 데이터 프라이버시 | 명시적 동의 기반 수집, 가명화 처리 후 파트너 은행과 공유 |
