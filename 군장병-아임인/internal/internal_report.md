## Section 1. 시장 규모 분석 (TAM / SAM / SOM)

### 1.1 TAM (Total Addressable Market): 전체 군 장병 시장

```
TAM 계산식
─────────────────────────────────────────────────
전체 현역 장병 수: 365,000명 (2024 국방통계포털)
평균 가처분 소득: 월 150~205만원 (병장 기준)
잠재 납입 가능액: 월 30만원 (가처분 소득의 15~20%)
연간 총 납입 규모: 365,000명 × 30만원 × 12개월
                = 1조 3,140억원/년

→ TAM: 약 1조 3,000억원/년
   (플랫폼 수수료 1.1% 기준 최대 수수료 수익: 약 143억원/년)
─────────────────────────────────────────────────
```

> ℹ️ **주석:** TAM은 이론적 상한선입니다. 군인공제회·장병내일준비적금 등 기존 상품과의 시장 분할을 감안해야 합니다.

## Section 3. 전략 평가: Do's & Don'ts

### 3.1 최악의 전략 (Worst Strategy — 절대 피해야 할 접근)

#### ❌ WORST 1: 규제 미비 상태에서 자체 영업 시도

- **행위:** 국방부·금감원 승인 없이 군 내부에서 자체적으로 마케팅 활동
- **리스크:** 
  - 국방부 '부대 내 영리 행위 제한' 규정 위반 → 전면 퇴출
  - 언론 보도 시 "군인 상대 불법 영업" 프레임 → 브랜드 치명타
  - 금감원 무인가 영업 조사 → 법적 제재
- **왜 유혹적인가:** 빠른 사용자 확보 욕심 + 공식 채널 구축의 느린 속도

#### ❌ WORST 2: 국방부를 '을'로 대하는 접근

- **행위:** "우리가 먼저 해주겠다"는 식의 일방적 제안
- **리스크:**
  - 국방부 관료 조직의 반감 → 검토 지연 또는 거절
  - 금융 사고 발생 시 모든 책임을 티웨이브에 전가하는 구조
- **올바른 접근:** 국방부를 **공동 이해관계자**로 설정 → "함께 정책을 만든다"는 프레임

#### ❌ WORST 3: 전군 즉시 확산 목표 설정

- **행위:** 파일럿 없이 36만명 전체 대상 런칭 시도
- **리스크:**
  - 운영 미숙으로 인한 대규모 장애 → 군 내 신뢰 완전 붕괴
  - 보안 사고 발생 시 국가 보안 문제로 확대
  - 회복 불가능한 브랜드 손상
- **올바른 접근:** 1만명 → 5만명 → 18만명 → 36만명 단계적 확산

#### ❌ WORST 4: 투자 수익률 강조 마케팅

- **행위:** "아임인으로 돈 불리세요" 식의 수익률 중심 메시지
- **리스크:**
  - 금감원 '유사수신행위' 오해 → 규제 조사
  - 국방부의 "고위험 투자 조장" 우려 → 협력 거절
- **올바른 접근:** "안전한 저축 습관" + "목돈 마련" 프레임으로만 커뮤니케이션

---

### 3.2 최선의 전략 (Best Strategy — 핵심 레버리지)

#### ✅ BEST 1: 금감원 샌드박스 선점 (가장 중요한 첫 번째 수)

- **행위:** 군 장병 특화 ROSCA 서비스로 혁신금융서비스 신청
- **효과:**
  - 법적 불확실성 제거 → 국방부의 협력 신뢰 기반 확보
  - "금감원 공인 혁신금융" 타이틀 → 마케팅 자산
  - 규제 특례 확보 → 경쟁자 진입 장벽
- **타임라인:** MOU 체결과 동시 진행 (M+1)

#### ✅ BEST 2: 국방부 내 '금융 교육' 예산으로 진입

- **행위:** 아임인을 '서비스 판매'가 아닌 '금융 교육 솔루션'으로 프레임화
- **효과:**
  - 영리 행위 제한 규정 우회 가능 (교육 프로그램은 허용)
  - 국방부 예산으로 파일럿 비용 일부 충당 가능
  - 금융위원회의 '군 금융교육 의무화' 정책과 자연스러운 연결
- **핵심 메시지:** "아임인은 체험형 금융 교육 플랫폼입니다"

#### ✅ BEST 3: 파일럿 부대 지휘관을 내부 챔피언으로 육성

- **행위:** 파일럿 사단장·연대장에게 사업 취지와 기대효과 상세 브리핑
- **효과:**
  - 내부 채널을 통한 자발적 확산 (Top-Down 지지)
  - 지휘관이 직접 장병에게 "이거 써봐라" → 가입률 극대화
  - 성공 시 해당 지휘관의 경력 자산화 → 적극적 지지
- **실행:** 지휘관 전용 대시보드 제공 (부대 저축 현황 실시간 모니터링)

#### ✅ BEST 4: '전역자 커뮤니티' 역방향 바이럴 전략

- **행위:** 파일럿 전역자들의 성공 스토리를 군 관련 커뮤니티(에브리타임·보배드림)에 확산
- **효과:**
  - 입대 예정자들의 "입대 후 아임인 써야겠다" 사전 인지 확보
  - 자발적 오프라인 바이럴 → 광고비 최소화
  - 국방부에 "장병들이 원하는 서비스"라는 수요 근거 제공

#### ✅ BEST 5: 군인공제회와의 협력 구조 구성

- **행위:** 군인공제회와 공동 상품 개발 (군인공제회 금리 보장 + 아임인 플랫폼 운영)
- **효과:**
  - 기존 공적 금융 기관의 신뢰 레버리지
  - "경쟁자" 아닌 "파트너" → 국방부 설득 용이
  - 군인공제회의 약점(디지털 UX) = 아임인의 강점 → 시너지

## Section 4. 법적·규제적 리스크 분석 및 대응

### 4.1 금융감독원 규제 리스크

#### 4.1.1 주요 규제 리스크 매트릭스

| 규제 항목 | 리스크 수준 | 발생 가능성 | 대응 방안 |
|-----------|:----------:|:-----------:|-----------|
| **유사수신행위법** | 높음 | 중간 | 원금 보장 금지 명시 + 에스크로 분리 보관 |
| **전자금융거래법** | 중간 | 낮음 | 전자금융업 등록 완료 (이미 충족) |
| **P2P 금융법 (온투업법)** | 낮음 | 낮음 | ROSCA 구조 = P2P 대출 아님 (법적 구분 명확) |
| **개인정보 보호법** | 높음 | 중간 | 군번 기반 익명화 + 목적 외 사용 금지 |
| **스트레스 DSR 규제** | 낮음 | 없음 | 아임인 = 대출 아닌 저축 → DSR 적용 없음 |

#### 4.1.2 유사수신행위법 리스크 상세

유사수신행위법은 **"원금 이상을 보장하는 수신 행위"** 를 금지합니다. ROSCA 구조는 원금 보장을 하지 않으므로 이 법의 적용을 받지 않습니다. 단, 마케팅 문구에서 "원금 보장" "확정 수익" 등의 표현을 절대 사용하지 않아야 합니다.

```
안전 문구 (O): "그룹 납입금 합산 수령", "목돈 마련", "저축 습관"
위험 문구 (X): "원금 보장", "확정 이자", "연 X% 수익 보장"
```

#### 4.1.3 금융규제 샌드박스 신청 전략

| 단계 | 내용 | 타임라인 |
|------|------|----------|
| 사전 상담 | 금융위원회 핀테크 혁신과 면담 | M+1 |
| 혁신금융서비스 신청서 제출 | 군 장병 특화 ROSCA 서비스 | M+2 |
| 심의 (평균 60일) | 금융위원회 혁신금융심사위원회 | M+2~M+4 |
| 지정 및 특례 적용 | 최대 2년 (연장 가능) | M+4 |

> ✅ **샌드박스 신청 핵심 논거:**
> "군 장병 빚투 문제(444억원) 해결을 위한 공공-민간 협력 혁신 금융 서비스"
> → 사회적 가치 명확 → 심사 통과 가능성 높음

### 4.2 국방부 규제 리스크

#### 4.2.1 주요 규제 리스크 매트릭스

| 규제 항목 | 리스크 수준 | 발생 가능성 | 대응 방안 |
|-----------|:----------:|:-----------:|-----------|
| **부대 내 영리 행위 금지** | 높음 | 높음 | 국방부 공식 복지 프로그램 등재를 통해 '비영리 목적 복지 서비스'로 분류 |
| **군사 정보 보안 (망분리)** | 높음 | 중간 | 군 인트라넷 미연결 + 공인 클라우드 보안 인증(CSAP) 획득 |
| **개인정보 (군번·위치 등)** | 높음 | 낮음 | 최소한의 개인정보만 수집 + 군번 대신 고유 익명 ID 사용 |
| **장병 금융 자율성 침해** | 중간 | 낮음 | 100% 자발적 가입 + 이탈 자유 보장 (강제 납입 없음) |
| **수익 배분 논란** | 낮음 | 중간 | 수수료 구조 국방부에 완전 공개 + 감사 허용 |

#### 4.2.2 군사 보안 기준 충족을 위한 기술 설계

| 보안 요건 | 충족 방법 | 인증 기준 |
|-----------|-----------|-----------|
| 데이터 서버 국내 위치 | AWS 서울 리전 또는 KT 클라우드 전용 | CSAP(클라우드 보안 인증) |
| 망분리 | 군 인트라넷과 완전 분리된 공개 인터넷 경로만 사용 | 국방부 보안감사 통과 |
| 개인정보 암호화 | AES-256 암호화 + 군번 미수집 | 개인정보보호 인증(ISMS-P) |
| 접근 통제 | 장병 본인 생체인증 + OTP 2단계 | FIDO2 표준 |
| 감사 로그 | 불변 저장 (블록체인 기반 감사 추적) | 국방부 감사팀 공유 |

#### 4.2.3 '영리 행위 금지' 규정 우회 전략

국방부 훈령 상 부대 내 영리 행위는 금지되어 있습니다. 그러나 **공식 복지 프로그램으로 지정된 서비스**는 예외입니다.

```
전략:
아임인 → "국방부 공식 장병 금융 복지 플랫폼"으로 지정 선취
          ↓
금융 교육 + 저축 지원 기능 강조 (영리 목적 최소화)
          ↓
수수료는 서비스 운영 비용으로 투명 공개 (영리 행위 이미지 차단)
          ↓
MOU 체결 시 국방부 감사 권한 포함 → 신뢰 극대화
```

## Section 6. 경쟁 환경 및 선점 필요성

### 6.1 잠재 경쟁자 분석

| 경쟁자 | 위협 수준 | 군 진입 가능성 | 아임인 우위 |
|--------|:--------:|:-------------:|------------|
| **토스** | 높음 | 중간 (이미 일부 진입) | ROSCA 특화 vs 일반 핀테크 |
| **카카오페이** | 높음 | 중간 | 저축 특화 vs 결제 중심 |
| **케이뱅크·카카오뱅크** | 중간 | 낮음 | 소셜 금융 vs 단순 예금 |
| **국방부 자체 앱** | 중간 | 중간 | 민간 혁신 속도 vs 관료적 개발 |
| **해외 핀테크** | 낮음 | 낮음 | 국내 법규 준수 강점 |

### 6.2 선점의 가치

군 채널은 **독점적 공식 파트너십 구조**로 인해 선점 효과가 매우 큽니다.

```
선점 시나리오 (아임인 선점):
   국방부 MOU → 공식 복지 프로그램 등재
   → 후발 경쟁자 진입 시 "동일 목적 중복 서비스" → 국방부 거절 가능

후발 진입 시나리오 (경쟁자 선점):
   경쟁사가 먼저 MOU → 아임인 진입 기회 대폭 축소
   → 차별화된 기능으로 별도 승인 필요 (훨씬 어려움)
```

> ⚠️ **경쟁 긴박성:** 토스·카카오페이는 이미 군인 대상 서비스를 운영 중입니다. 그러나 아직 **ROSCA 기반 저축 특화 군 복지 프로그램을 MOU 수준으로 진행한 민간 기업은 없습니다.** 지금이 선점 창문(Window of Opportunity)입니다.
